# 🚀 AI 트레이딩 팩토리 (2-PC) 운영 및 문제 해결 가이드

이 문서는 Antigravity AI와 다온님이 함께 구축한 2-PC 기반 트레이딩 시스템의 운영 방법과 구글 드라이브 동기화 셋업을 기록한 문서입니다. 재부팅 후에도 언제든 이 파일을 열어 참고하실 수 있습니다.

---

## 1. 2-PC 역할 분담 요약

*   **🖥️ PC 1 (실전 매매 및 메인 서버)**
    *   **역할:** 키움증권 실전투자 로그인, 텔레그램 컨트롤러 실행, 주식/선물 봇 라이브 매매
    *   **경로:** `c:\antigravity\노트븍활용`
    *   **설정:** `config.json`에서 `"environment": "live"` 설정

*   **💻 PC 2 (AI 연구소 및 모의투자 서버)**
    *   **역할:** 키움증권 모의투자 로그인, 야간 백테스트(K값 최적화) 무한 루프 가동, Claude Code 에이전트 가동
    *   **경로:** `c:\antigravity\노트븍활용`
    *   **설정:** `config.json`에서 `"environment": "mock"` 설정

---

## 2. 구글 드라이브 동기화 세팅 (핵심)

과거 노트북 C드라이브 전체를 동기화하려다 발생한 문제를 해결하기 위해 시스템을 개선했습니다.

1.  파이썬 코드 자체는 구글 드라이브로 동기화하지 않습니다. (에러 유발 방지 -> 코드는 GitHub으로 관리)
2.  대신, **AI가 백테스트한 결과물(파라미터)만** 구글 드라이브를 통해 교환합니다.
3.  현재 파이썬 코드는 `G:\내 드라이브\AI_Trading_Data` 폴더를 직접 바라보도록 수정되어 있습니다.
4.  **확인 사항:** PC 1과 PC 2 모두 **구글 드라이브 데스크톱**이 설치되어 있고, `G:` 드라이브로 정상 인식되는지만 켜두시면 완벽하게 텔레그램 핫-리로드(전략 자동 교체)가 작동합니다.

---

## 3. 키움증권 "암호화 키교환 실패" 에러 대처법

봇을 실행했는데 키움증권 로그인 창 대신 "암호화 키교환 실패" 창이 뜬다면, 키움증권의 자체 업데이트가 꼬인 것입니다.

1.  PC를 **재부팅** 하거나, 백그라운드의 키움 프로세스(`opstarter.exe`, `ncstarter.exe` 등)를 강제 종료합니다.
2.  파이썬 봇을 켜기 **전에**, 윈도우 시작메뉴에서 **KOA StudioSA (또는 영웅문)**를 직접 실행합니다.
3.  직접 아이디/비밀번호를 치고 로그인하여 화면 중간에 뜨는 **"버전처리 중입니다..."** 다운로드가 끝날 때까지 기다립니다.
4.  정상적으로 창이 열리면 끝난 것입니다. KOA Studio를 끄고, 다시 텔레그램 봇(파이썬)을 실행하시면 됩니다.

## 4. 📱 텔레그램 주요 명령어 리마인드

*   `!상태` : 봇 가동 상태 확인
*   `!긴급정지` : 모든 주식/선물 시장가 매도 후 프로세스 강제 종료
*   `!최적화결과` : PC 2(AI 랩)가 구글 드라이브에 저장해둔 최신 파라미터(K값) 브리핑 받기
*   `!전략승인` : AI가 추천한 1위 전략을 실전 봇(PC 1)에 무중단(Hot-reload)으로 즉시 적용하기
